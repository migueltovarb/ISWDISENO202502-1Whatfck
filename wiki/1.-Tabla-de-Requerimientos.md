# Tabla de Requerimientos

## Requerimientos Funcionales

| ID | Descripción | Prioridad | Estado |
|----|-------------|-----------|--------|
| RF1 | El sistema debe permitir a los usuarios registrarse con nombre, email y contraseña. | Alta | Pendiente |
| RF2 | El sistema debe permitir a los usuarios autenticarse. | Alta | Pendiente |
| RF3 | El sistema debe permitir registrar un nuevo reclamo con título, descripción, tipo de servicio y ubicación. | Alta | Pendiente |
| RF4 | El sistema debe mostrar en el dashboard del cliente una lista de sus solicitudes activas al iniciar sesión. | Alta | Pendiente |
| RF5 | El sistema debe mostrar el detalle de un reclamo con un timeline de mensajes y respuestas para clientes. | Media | Pendiente |
| RF6 | El sistema debe enviar notificaciones por email cuando el estado del reclamo cambie. | Media | Pendiente |
| RF7 | El sistema debe permitir a usuarios de soporte ver lista de reclamos pendientes. | Alta | Pendiente |
| RF8 | El sistema debe permitir a usuarios de soporte responder a reclamos con soluciones. | Alta | Pendiente |
| RF9 | El sistema debe permitir a usuarios de soporte marcar reclamos como resueltos. | Media | Pendiente |
| RF10 | El sistema debe mostrar el detalle completo de un reclamo con hilo de conversación para soporte y administradores. | Media | Pendiente |
| RF11 | El sistema debe cerrar automáticamente un reclamo si no hay respuesta del cliente en 3 días después de una respuesta de soporte. | Media | Pendiente |
| RF12 | El sistema debe permitir a administradores cambiar el estado de cualquier reclamo. | Alta | Pendiente |
| RF13 | El sistema debe permitir a administradores cerrar reclamos definitivamente. | Alta | Pendiente |
| RF14 | El sistema debe permitir a administradores archivar reclamos para ocultarlos del usuario sin eliminarlos de la base de datos. | Media | Pendiente |

## Requerimientos No Funcionales

| ID | Descripción | Prioridad | Estado |
|----|-------------|-----------|--------|
