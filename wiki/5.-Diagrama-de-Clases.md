# Diagrama de Clases

```mermaid
classDiagram
    Usuario <|-- Cliente
    Usuario <|-- Soporte
    Usuario <|-- Administrador
    Cliente "1" *-- "*" Reclamo : registra
    Reclamo "1" *-- "*" Respuesta : tiene
    Soporte "1" o-- "*" Respuesta : envia
    Reclamo "1" *-- "*" Notificacion : genera
    Administrador "1" o-- "*" Reclamo : gestiona


    class Usuario{
        +int id
        +String nombre
        +String email
        +String password
        +String tipo
        +registrar()
        +iniciarSesion()
    }

    class Cliente{
        +verMisReclamos()
    }

    class Soporte{
        +responderReclamo()
        +marcarResuelto()
    }

    class Administrador{
        +cambiarEstado()
        +cerrarReclamo()
        +archivarReclamo()
    }

    class Reclamo{
        +int id
        +String titulo
        +String descripcion
        +String estado
        +int usuarioId
        +Date fechaCreacion
        +registrar()
        +consultarEstado()
        +cerrarAutomatico()
    }

    class Respuesta{
        +int id
        +String mensaje
        +int reclamoId
        +int soporteId
        +Date fecha
        +enviar()
    }

    class Notificacion{
        +int id
        +String mensaje
        +int reclamoId
        +enviar()
    }
```

## Descripción
- **Usuario:** Clase base con atributos comunes.
- **Cliente:** Usuario que registra reclamos.
- **Soporte:** Usuario que responde y marca reclamos como resueltos.
- **Administrador:** Usuario con permisos avanzados (cambiar estados, cerrar, archivar).
- **Reclamo:** Ticket principal con estados y cierre automático.
- **Respuesta:** Mensajes de soporte a reclamos.
- **Notificacion:** Alertas por email.
