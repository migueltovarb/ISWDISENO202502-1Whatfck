# Escenario de Calidad

## Escenario 1: Rendimiento - Tiempo de Respuesta

- **Estímulo:** Un usuario registra un nuevo reclamo durante horas pico.
- **Ambiente:** Sistema con 500 usuarios concurrentes.
- **Respuesta:** El sistema confirma el registro en menos de 2 segundos.
- **Medida de Respuesta:** Tiempo de respuesta promedio < 2s, con un percentil 95 < 5s.

## Escenario 2: Seguridad - Autenticación

- **Estímulo:** Un usuario intenta acceder sin credenciales válidas.
- **Ambiente:** Cualquier momento.
- **Respuesta:** El sistema deniega el acceso y registra el intento.
- **Medida de Respuesta:** 100% de denegación para accesos no autorizados.

## Escenario 3: Usabilidad - Registro de Reclamo

- **Estímulo:** Un usuario nuevo intenta registrar un reclamo.
- **Ambiente:** Interfaz web.
- **Respuesta:** El usuario completa el formulario en menos de 5 minutos sin ayuda.
- **Medida de Respuesta:** Tasa de éxito > 90% para usuarios sin experiencia previa.

## Escenario 4: Escalabilidad - Carga Máxima

- **Estímulo:** El número de usuarios concurrentes alcanza 1000.
- **Ambiente:** Durante un evento de mantenimiento masivo.
- **Respuesta:** El sistema mantiene funcionalidad sin degradación significativa.
- **Medida de Respuesta:** Disponibilidad > 99%, tiempo de respuesta < 5s.

## Escenario 5: Confiabilidad - Recuperación de Fallos

- **Estímulo:** Ocurre una falla en el servidor de base de datos.
- **Ambiente:** Operación normal.
- **Respuesta:** El sistema se recupera automáticamente en menos de 1 minuto.
- **Medida de Respuesta:** Tiempo de inactividad < 1 min, pérdida de datos cero.
